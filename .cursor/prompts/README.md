# 📋 Шаблоны промптов для Cursor

## Как использовать
Сохрани эти шаблоны в `.cursor/prompts/` и вызывай через `@prompts/название` или копируй в чат.

## 1️⃣ Стратегия и валидация гипотез

**Файл:** `.cursor/prompts/01-strategy.prompt`

```
[СТРАТЕГ] Проверь гипотезу для проекта «Надо»

[Контекст]
- Ниша: {{ниша, например "автомойки самообслуживания"}}
- Боль: {{описание проблемы}}
- Гипотеза: {{предполагаемое решение}}
- География: Владивосток (часовой пояс +10)
- ЦА: Владельцы малого автобизнеса (1–10 сотрудников)

[Задача]
1. Проверь гипотезу: насколько она решает боль?
2. Предложи 3 варианта реализации (от простого к сложному)
3. Оцени: сложность (низкая/средняя/высокая), время (недели), потенциальный эффект (₽/мес)
4. Сформулируй метрику успеха для теста
5. Назови риски и как их минимизировать

[Формат]
- Краткий вердикт (да/нет/с доработками)
- Таблица вариантов с оценками
- Рекомендация + следующий шаг
- Вопросы на уточнение (если нужно)
```

## 2️⃣ Проектирование архитектуры

**Файл:** `.cursor/prompts/02-architecture.prompt`

```
[АРХИТЕКТУРА] Спроектируй компонент для проекта «Надо»

[Контекст]
- Модуль: {{например "orders" или "clients"}}
- Стек: FastAPI + SQLAlchemy + PostgreSQL + JSONB
- Требования: 152-ФЗ, soft delete, индексы, миграции

[Задача]
1. Спроектируй схему БД (таблицы, поля, типы, индексы, FK)
2. Нарисуй диаграмму Mermaid (ER-диаграмма)
3. Опиши API эндпоинты (REST, версия v1)
4. Укажи бизнес-валидацию и ограничения
5. Предложи структуру миграций (Alembic)

[Обязательно]
- Soft delete (`deleted_at`)
- Audit log (`created_at`, `updated_at`, `created_by`)
- Индексы по фильтрам и внешним ключам
- JSONB для кастомных полей (если нужно)
- Поддержка японских номеров авто (если применимо)

[Формат]
1. SQL схема (CREATE TABLE)
2. Mermaid диаграмма
3. Список эндпоинтов (метод, путь, описание)
4. Примеры запросов/ответов (JSON)
5. Миграция (alembic upgrade/downgrade)
```

## 5️⃣ Code Review и безопасность

**Файл:** `.cursor/prompts/05-review.prompt`

```
[РЕВЬЮ] Проверь код на уязвимости и best practices

[Контекст]
- Файл/модуль: {{путь к файлу или @file}}
- Язык: {{Python/TypeScript}}
- Критичность: {{критично для прода / рефакторинг}}

[Чек-лист проверки]
1. Безопасность:
   - SQL injection (параметризованные запросы)
   - XSS (sanitization входных данных)
   - CSRF (токены для мутаций)
   - Авторизация (проверка прав доступа)
   - Секреты (нет хардкода паролей/ключей)

2. Производительность:
   - N+1 проблема (eager loading)
   - Индексы БД
   - Кэширование (где применимо)
   - Пагинация больших списков

3. Надёжность:
   - Обработка ошибок (try/catch, логирование)
   - Graceful degradation
   - Retry logic (для внешних API)

4. Читаемость:
   - Именование переменных/функций
   - Комментарии (зачем, а не что)
   - DRY (нет дублирования)
   - Размер функций (<50 строк)

[Формат]
- Список найденных проблем (критичность: high/medium/low)
- Конкретные строки кода
- Предложенные фиксы (код в блоке)
- Рекомендации по рефакторингу
```

```
[СТРАТЕГ] Проработай пользовательский сценарий для проекта «Надо»

[Формат]
```

## 💡 Советы по использованию

1. **Всегда указывай режим** в начале запроса: `[СТРАТЕГ]`, `[АРХИТЕКТУРА]`, `[КОД]` и т.д.

2. **Используй @file ссылки** в Cursor для контекста:
   - `@src/backend/app/api/v1/orders.py` — конкретный модуль
   - `@docs/03-mvp-spec.md` — спецификация
   - `@AGENTS.md` — правила проекта
   - `@.ai/memory/` — долговременная память

3. **Для многофайловых задач** используй Composer (`Cmd/Ctrl+K`):
   ```
   Создай модуль orders с моделями, API, сервисами и тестами. 
   Используй паттерн Repository. Стек: FastAPI + SQLAlchemy + pytest.
   ```

4. **Для рефакторинга** используй Agent Mode (`Cmd/Ctrl+Shift+I`):
   ```
   Оптимизируй запросы к БД в модуле clients. Добавь индексы, исправь N+1 проблему.
   ```

5. **Сохраняй важные решения** в `.ai/memory/` (см. шаблон в `.ai/memory/README.md`):
   ```bash
   # Создай файл в нужной категории (business / tech / design / iterations):
   # .ai/memory/tech/2026-05-08_orders-schema.md
   ```
