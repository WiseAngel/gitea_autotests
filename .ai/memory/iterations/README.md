# 🔄 Логи Итераций

Эта директория отслеживает итерации разработки и прогресс спринтов для QA фреймворка.

## Назначение

Логи итераций предоставляют:
- Хронологическую запись изменений
- Sprint goals и outcomes
- Technical debt tracking  
- Lessons learned
- Decision history с контекстом

## Соглашение по именованию файлов

Используй ISO date format для документов итераций:
```
YYYY-MM-DD_HH-MM_description.md
```

Пример: `2026-05-12_11-02_ci-fixes-and-refactoring.md`

## Шаблон структуры (формат [ПАМЯТЬ])

```markdown
# [ПАМЯТЬ] Сессия: Описание сессии

## Дата сессии
YYYY-MM-DD HH:MM
Краткое описание контекста сессии.

## Цель сессии
1. Цель 1
2. Цель 2
3. Цель 3

## Выполненные действия

### 1. Группа задач 1
- **Проблема**: Описание проблемы
- **Решение**: Описание решения
- Детали выполнения

### 2. Группа задач 2
- **Проблема**: Описание проблемы  
- **Решение**: Описание решения
- Детали выполнения

## Итоговый статус
- ✅ Все проверки линтера проходят успешно
- ✅ Тесты запускаются без ошибок
- ✅ Документация актуализирована

## Файлы, подвергшиеся изменениям
1. `файл1.py` (создан/изменён) - описание
2. `файл2.py` (создан/изменён) - описание
3. `файл3.py` (создан/изменён) - описание

## Рекомендации для будущих сессий
- Рекомендация 1
- Рекомендация 2
- Рекомендация 3
```

---

> 💡 **Совет**: Просматривай предыдущие итерации перед началом новой работы для сохранения непрерывности и избежания повторения ошибок.
