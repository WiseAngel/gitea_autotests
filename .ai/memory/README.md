# 🧠 Память проекта (.ai/memory)

Эта папка — долговременная «память» проекта между сессиями. Здесь живут **окончательные** решения, которые AI-ассистент или человек должен помнить и учитывать в дальнейшей работе.

> ⚠️ Это не то же самое, что `artifacts/`.
> - `artifacts/` — рабочая площадка: размышления (черновики), макеты, прототипы, ADR в процессе.
> - `.ai/memory/` — крайний слой: то, что уже принято и не должно теряться. Обновляется по факту, не по итерации.

---

## 📂 Структура

| Папка | Что хранит | Примеры файлов |
|-------|-----------|----------------|
| `business/` | Бизнес-решения, экономика, ниши | `2026-05-08_pricing-final.md`, `2026-05-12_first-niche-autoservice.md` |
| `tech/` | Архитектура, выбор стека, схемы БД, API-контракты | `2026-05-08_orm-choice.md`, `2026-05-10_rls-strategy.md` |
| `design/` | UI/UX правила, цветовая система, паттерны | `2026-05-08_color-palette.md`, `2026-05-10_kanban-pattern.md` |
| `iterations/` | Лог итераций (что менялось, почему) | `2026-05-08_iter-1-mvp-scope.md` |
| `backlog/` | Отдельные плановые задачи и будущие шаги | `2026-05-26_bot-infrastructure-and-first-run.md` |

## 📝 Шаблон файла памяти

```markdown
# Решение: <короткое название>

**Дата**: 2026-05-08
**Категория**: tech | business | design | iterations
**Статус**: Принято / Пересмотрено / Устарело

## Контекст
Что обсуждалось, почему встал вопрос?

## Решение
Что именно решили? Кратко и однозначно.

## Альтернативы
- A: ... (отклонено, потому что ...)
- B: ... (отклонено, потому что ...)

## Последствия
- ✅ Плюсы: ...
- ⚠️ Минусы: ...
- 🔧 Технический долг: ...

## Связанные файлы
- `artifacts/decisions/adr-XXX.md`
- `src/backend/app/models/...`
- `docs/03-mvp-spec.md`

## История изменений
- 2026-05-08: создано
- 2026-05-15: пересмотрено (см. iterations/2026-05-15_*.md)
```

## 🔄 Workflow

1. **Идея/обсуждение** → `artifacts/thinking/` (черновики)
2. **Прототип/макет** → `artifacts/pages/` или `artifacts/mockups/`
3. **Формальное решение (ADR)** → `artifacts/decisions/`
4. **Закрепление в памяти** → файл в `.ai/memory/<категория>/`
5. **Реализация** → код + ссылка в коде на файл памяти

---

## 🤖 Использование с разными AI-ассистентами

Папка работает с любыми AI-инструментами (Cursor, Claude Code, Codex, Aider, Continue и т.д.). Способ использования:

- При начале сессии: **попросить ассистента прочитать `.ai/memory/` целиком** или релевантную подкатегорию.
- При важном решении: **попросить ассистента обновить или создать файл памяти**.
- В `AGENTS.md` уже прописано правило ссылаться на эту папку.

---

## 📌 Правила именования

- Формат: `YYYY-MM-DD_kebab-case-topic.md`
- Категория = подпапка (не дублировать в имени файла)
- Если решение пересмотрено — старый файл остаётся (история), новый ссылается на него
- Для `backlog/` используй тот же формат, но храни там только задачи, которые ещё не стали окончательным решением
