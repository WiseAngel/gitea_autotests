## Контекст
Фреймворк Playwright уже стабилен и CI зелёный. Следующий практический шаг — довести интеграцию с `sdet-orchestrator` до рабочего состояния, чтобы запускать тесты через Telegram-бота.

## Цель backlog-задачи
1. Поднять инфраструктуру бота без Docker Desktop пока это невозможно на Win10
2. Подготовить запуск первого `/run` через Telegram-бота
3. Сохранить сценарий как отдельный ориентир для будущей сессии

## Трек 1 — Поднять инфраструктуру бота
- **Блокер**: Docker Desktop сейчас недоступен на Win10, поэтому бот нельзя запустить в полном стекe.
- **Что нужно сделать, когда Docker станет доступен**:
  1. Запустить Docker Desktop
  2. Создать `docker-compose.yml` в `infra/`
  3. Создать `.env` на основе `.env.example`
  4. Поднять Redis и PostgreSQL через compose
  5. Выполнить `alembic upgrade head`
  6. Запустить бот и настроить ngrok для webhook

## Трек 3 — Первый `/run` через бота
- **Целевая цепочка**:
  `/run → main → staging → smoke`
- **Ожидаемый результат**:
  - `passed: ~13`
  - `failed: 0`
  - `skipped: ~8`
- **Цель**: подтвердить, что `sdet-orchestrator` реально запускает Playwright suite и получает результат end-to-end

## Почему это важно
- Даёт полноценную проверку цепочки "бот → запуск → отчёт"
- Делает проект сильнее для портфолио: есть не только CI, но и внешний оркестратор
- Закрывает ключевой демонстрационный сценарий для senior/lead QA automation
